Microservices Details

Microservices Overview

An overview of the microservices:

User Interface

Description:The goal of this component is to serve the user:

  • Log In
  • Register
  • Create Alarm
  • Add Favourite Coin
  • See Crypto Coins
  • See News
  • See Predictions

Github repository:frontend

Kafka topics consumed from:None

Kafka topics produced to:None

Events reacted to:Server push notification

Events produced:None

EDA Patterns implemented:None

Build

To build this microservice you need to:

  • Run "docker build -t frontend ."

Run

To run this microservice you need to:

  • A: Run docker with docker-compose up on the main directory to build the whole project
  • B: Have NodeJS installed and run "npm start" in the main front-end folder

Account microservice

Description:The goal of this component is to store user data and serve the main user functionalities:

  • Handle Registration and log in security
  • Store user alarms
  • Push Notifications to front-end by SSE when an alarm becomes active
  • Verify if alarm becomes true (each 10 seconds)
  • Store user favourite coins
  • Interface with email to notify user

Github repository:accountApp

Kafka topics consumed from:alarmResponse

Kafka topics produced to:alarmRequest

Events reacted to:None

Events produced:Alarm Verification, Alarm notification

EDA Patterns implemented:CQRS

REST API endpoints:

  • /addAlarm > takes the logged in user and adds an alarm that comes in json format
  • /getAlarm > takes the logged in user and returns all alarms
  • /deleteAlarm={id} > takes an alarm id and sends a json object aswell to delete the alarm with that id from user
  • /getFavouriteCoins > takes the logged in user and returns his favourited coins
  • /addFavouriteCoins > takes the logged in user and adds a coin in a json format
  • /deleteFavouriteCoin={id} > takes the logged in user and a favourited coin id and deletes it
  • /api/v1/login > takes json object (user,password)
  • /api/v1/registration > takes json object (user,password) and stores the user
  • /api/v1/registration/confirm > takes a registration token and verifies a user after registration

Build

To build this microservice you need to:

  • Run "docker build -t account ."

Run

To run this microservice you need to:

  • A: Run docker with docker-compose up on the main directory to build the whole project
  • B: Build project with mvn clean and mvn build then run the JAR

Historic microservice

Description:The goal of this component is to store cryptocurrencies information and communicate with the machine learning module:

  • Give users overall cryptocurrencies list
  • Gives detailed information on coins
  • Confirms if an alarm is true
  • Communicates the ML predictions to the frontend

Github repository:History

Kafka topics consumed from:CoinsLiveUpdate,historic,alarmRequest,MLPredictions

Kafka topics produced to:MLCoinInfo, alarmResponse

Events reacted to:Alarm verification

Events produced:None

EDA Patterns implemented:CQRS

REST API endpoints:

  • /api/coins > returns all the cryptocurrencies
  • /api/historic={coinID} > returns historic information up to 30 days from a cryptocoin
  • /api/prediction/historic={CoinID} > returns machine learning prediction information up to 30 days from a cryptocoin

Build

To build this microservice you need to:

  • Run "docker build -t historic ."

Run

To run this microservice you need to:

  • A: Run docker with docker-compose up on the main directory to build the whole project
  • B: Build project with mvn clean and mvn build then run the JAR
